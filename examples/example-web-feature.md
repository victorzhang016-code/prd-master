# 智能客服机器人

**版本** v1.0　**日期** 2025-04-11　**状态** 待评审

---

## 一、为什么做这个

**现在的问题**：客服团队每天处理 2000+ 咨询，其中 60% 是重复性问题（订单查询、退换货政策、运费标准），人工回复平均耗时 5 分钟，用户等待时间长，客服离职率高达 30%。

**我们要做什么**：上线 AI 智能客服机器人，自动回复高频问题，复杂问题无缝转人工。

**做完之后**：用户的问题能在 10 秒内得到回复，客服可以专注于解决真正复杂的问题。

---

## 二、怎么做

### 核心流程

1. 用户点击客服入口
2. AI 机器人先应答，理解用户意图
3. 如果是高频问题 → 直接回复
4. 如果是复杂问题 → 询问「是否需要转人工」
5. 用户确认后 → 排队接入人工客服（带上下文）

### 关键细节

**意图识别**
- 覆盖 20 个高频意图（订单查询、退款申请、商品咨询等）
- 置信度 > 0.8 才自动回复，否则询问澄清

**转人工规则**
- 用户主动要求转人工
- 连续 3 次无法理解用户意图
- 涉及投诉、法律问题

**人机协作**
- 转人工时携带完整对话上下文
- 人工客服可以一键引用机器人回复

## 三、技术方案

| 方案 | 好处 | 坏处 | 结论 |
|------|------|------|------|
| 基于规则的关键词匹配 | 可控、成本低 | 扩展性差，理解能力弱 | ❌ |
| 调用 Claude API | 理解能力强，维护成本低 | 有 API 费用 | ✅ |
| 自研大模型 | 完全可控 | 成本极高，周期长 | ❌ |

**我们选方案 2**，因为业务验证阶段速度最重要，Claude API 的准确率已验证可达 90%+。

## 四、字段定义

| 字段 | 类型 | 说明 |
|------|------|------|
| `session_id` | string | 会话唯一ID |
| `user_id` | string | 用户ID |
| `message_type` | enum | user/bot/agent |
| `content` | text | 消息内容 |
| `intent` | string | 识别的意图 |
| `confidence` | float | 置信度 0-1 |
| `handoff_reason` | string | 转人工原因 |

## 五、边界情况

| 如果…… | 处理方式 |
|--------|---------|
| API 超时/失败 | 提示「服务繁忙」，提供「转人工」选项 |
| 用户发送图片 | 提示「暂不支持图片，请描述您的问题」 |
| 用户情绪激动（识别到关键词）| 立即提供「转人工」入口，不强制机器人处理 |
| 非工作时间 | 告知人工客服上班时间，机器人继续处理能回答的问题 |

**现状数据**：

| 指标 | 现在是多少 | 数据来源 |
|------|-----------|---------|
| 日均咨询量 | 2000+ | 客服系统 |
| 高频问题占比 | 60% | 工单分类统计 |
| 平均响应时间 | 5 分钟 | 客服系统 |
| 客服离职率 | 30%/年 | HR 数据 |

**假设与推算**：
- 假设：AI 能准确回答 70% 的高频问题
- 推算：每天减少 840 单人工处理（2000 × 60% × 70%），相当于节省 2-3 名客服人力

## 七、怎么算做好了

**北极星指标**：AI 解决率（AI 直接解决无需转人工的咨询占比）

| 指标 | 基准值 | 目标值 | 观测窗口 |
|------|--------|--------|---------|
| AI 解决率 | - | 50% | 上线后 30 天 |
| 用户满意度 | - | > 4.0/5 | 上线后 14 天 |
| 平均响应时间 | 5 分钟 | < 30 秒 | 上线后 7 天 |

**过程指标**：

| 过程指标 | 目标 | 说明 |
|---------|------|------|
| 意图识别准确率 | 90% | 机器人理解正确的比例 |
| 转人工率 | < 50% | 说明机器人能处理大部分问题 |
| 重复提问率 | < 20% | 用户重复同一问题的比例 |

**验收标准**：
- [ ] AI 解决率达到 50%
- [ ] 用户满意度不低于人工客服水平
- [ ] 无重大客诉（因机器人回复错误导致的投诉 < 1%）
